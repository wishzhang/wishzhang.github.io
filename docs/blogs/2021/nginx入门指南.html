nginx入门指南

仰望星空 2021/9/22 nginx

nginx (opens new window) 由一个主进程和一些工作进程 (opens new window)构成。主进程用来读取配置信息和维护工作进程，工作进程负责实际的请求处理。nginx 采用基于事件模型和依赖操作系统机制，来有效分发请求给工作进程。

# 开启，停用和重启

nginx 开启后可执行以下命令：

nginx -s signal
1

sginal:

  • stop 立即关掉
  • quit 正常关掉，等所有工作进程处理完手中的请求才关掉
  • reload 重新加载配置文件
  • reopen 重新打开日志文件

nginx 重新加载配置文件后，会根据配置文件开启新的工作进程，然后关掉旧的工作进程。
更多关于nginx信号的信息 (opens new window)

# 配置文件

nginx 的配置文件主要有两类指令来控制。指令分为简单指令和块指令。简单的指令由名称和参数组成，参数之间用空格分隔，并以分号结束。块指令与简单指令具有相同的结构，但它以一组由大括号包围的附加指令结束，而不是分号。一个块指令构成的直接作用域被称为上下文( events (opens new window) , http (opens new window) , server (opens new window) , location (opens new window) )。events 和 http 指令在 main (opens new window) 上下文中，server 在 http 中，location 在 server 中。

# 代理静态资源

web 服务器的一个重要的功能就是代理文件。接下来的一个例子是，将请求代理到不同的文件夹下。

location / {
	root /data/www;
}

location /images {
	root /data;
}

# 配置代理服务器

nginx 还有一个常用的功能是作为一个代理服务器来转发请求。我们将配置一个基本的代理服务器来接收图片文件的请求，然后将请求转发到被代理的服务器。在下面这个例子中，代理服务器和被代理服务器相关的配置都被定义在一个 nginx 实例上。

1）首先，被代理的服务器的配置如下，监听 (opens new window)8080端口：

server {
    listen 8080;
    root /data/up1;

    location / {
    }
}

这个服务器将监听8080端口，并将所有的请求映射到 /data/up1 文件目录下。在这个文件目录创建好 index.html 文件。
注意，这里只在 server 上下文环境中定义了 root (opens new window) , 而如果location里没有定义自己的 root，请求就会映射到 server 的 root。

2）接下来，配置代理服务器，主要是配置 proxy_pass (opens new window) 字段，配置如下：

server {
    location / {
        proxy_pass http://localhost:8080/;
    }

    location ~ \.(gif|jpg|png)$ {
        root /data/images;
    }
}
